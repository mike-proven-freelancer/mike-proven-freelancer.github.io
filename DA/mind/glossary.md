# Glossary

Terms introduced during the project (for AI/future context). Expand as teaching progresses.

| Term | Definition |
|------|------------|
| Benchmark dataset | Structured dataset built from multiple sources, used to evaluate predictive models and analytical frameworks. |
| Backtest | Evaluating a model or rule on historical data with a time-based split; compare predictions to actual outcomes. |
| Grain | Level of detail of a table (e.g., one row per post per day). |
| Dimension table | Table of entities (e.g., accounts, channels, creators); used for filtering and grouping. |
| Fact table | Table of events or metrics (e.g., daily performance, revenue); typically joined to dimensions. |
| Schema | Structure of tables: column names, types, and relationships. |
| Pipeline | Sequence of steps: ingest raw data → clean/transform → load into warehouse. |
| Synthetic data | Artificially generated data that mimics real patterns; no real individuals. |
